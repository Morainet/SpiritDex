# AI 模块 UI 重构 + 交互完善 实施计划

## 目标
解决 4 个问题：① 头部/标题区太平 ② 气泡/输入框粗糙 ③ 缺乏 AI 质感 ④ 三页风格不统一；并新增 4 项能力：① Chat 跳动修复 ② 结果结构化（精灵卡片）③ AI 能力导航/联动 ④ 多轮记忆 + 加载动画。

## 风格定位
**游戏化贴合主站**：精灵球红主色 + 属性色点缀 + 卡片化（`shadow-[var(--shadow-card)]` + `rounded-2xl`），AI 模块用紫/粉/青三色（illusion/cute/ice 属性色）区分问答/推荐/识别，统一成有设计语言的"AI 助手"模块。

## 改动清单

### Part A：后端改动（3 个文件，需重新打 jar）

**A1. ChatService 多轮记忆（`ai/ChatService.java`）**
- 新增重载 `streamChat(String question, List<ChatMessage> history, Consumer<List<Snippet>> onRefs)`
- 把 history 转成 Spring AI 的 `UserMessage`/`AssistantMessage` 拼到 prompt 前面（最近 4 轮 = 8 条消息）
- 用 Spring AI 的 `ChatMemory`（InMemoryChatMemory，session-id 用前端传的 `sessionId`，jar 里已有 `spring-ai-autoconfigure-model-chat-memory`）

**A2. AiChatController 接收历史（`controller/AiChatController.java`）**
- `ChatRequest` 加字段：`List<Msg> history`（每条 `{role, content}`）、`String sessionId`
- 透传给 `chatService.streamChat(question, history, sessionId, ...)`

**A3. RecommendService 结构化 JSON（`ai/RecommendService.java` + `Prompts.java`）**
- 改 prompt：要求 GLM 返回 JSON 数组 `[{slug, name, role, reason}]`（role=主力/辅助/对策，reason=推荐理由）
- 新增 `recommendStructured(...)` 返回 `Flux<RecommendItem>`（非流式 JSON，一次性返回——推荐内容短，流式体验不如一次性结构化卡片）
- AiRecommendController 加新端点 `POST /api/ai/recommend-cards` 返回 `Result<List<RecommendCard>>`
- 保留旧 `/recommend`（markdown 流式）兼容，新页用结构化端点

### Part B：前端 AI 能力导航 + 联动（2 个新文件 + 1 改）

**B1. 新建 `app/ai/page.tsx`**（AI 能力总览入口页）
- 三张大卡片入口：智能问答（紫）、阵容推荐（粉）、图片识别（青）
- 每张卡片用对应属性色渐变 + 图标 + 简介，点击进对应页
- 顶部 hero 区：渐变背景 + "灵宠 AI 助手" 标题

**B2. 改 `lib/nav-config.ts`**：AI 分组首项加 `/ai` 总览入口

**B3. ChatClient 底部加联动按钮**：问答页输入框下方加"🖼 识别精灵"、"⚔ 阵容推荐"快捷跳转链接

### Part C：三个 Client 组件 UI 重构（3 个文件重写）

**C1. `components/ChatClient.tsx`（问答）—— 修跳动 + 质感**
- **修跳动**：消息列表 `scrollTo` 改 `behavior: "auto"`（smooth 在流式高频更新时会抖），加 `overflow-anchor` CSS 稳定滚动锚点
- **AI 质感**：AI 消息加圆形头像（🤖 emoji + illusion 紫底），用户消息加头像（🧑 + surface-2 底）
- **加载动画**：流式输出时末尾加打字光标 `▋` 闪烁动画（CSS `@keyframes blink`）
- **气泡精修**：用户气泡用 primary 红，AI 气泡用 surface-2 + 左侧 3px illusion 紫色边条（区别于普通卡片）
- **多轮记忆**：维护 `history: {role, content}[]` state，每次 ask 时带上历史；显示"已记住 N 轮对话"小标识
- **头部**：输入区上方加 AI 助手身份条（头像 + "灵宠档案 AI · 基于全站数据"）

**C2. `components/RecommendClient.tsx`（推荐）—— 结构化卡片**
- 改调 `/api/ai/recommend-cards`（结构化 JSON）
- 结果区：5 张精灵卡片网格（用 PetCard 风格 + 属性色顶条 + 角色徽章 + 推荐理由），点击跳详情

**C3. `components/IdentifyClient.tsx`（识别）—— 结构化卡片**
- 候选精灵改用 PetCard 样式（属性色顶条 + 头像 + 相似度进度条），不用现在的简陋 border 卡
- 加载态：上传后显示"AI 识别中…" + 旋转动画
- 头部：渐变 hero（ice 青）+ 图标

## 关键技术决策
1. **ChatMemory 用 InMemory**：单机够用，Phase 7 账号体系再换持久化。session-id 前端用 `crypto.randomUUID()` 存 localStorage
2. **推荐改非流式 JSON**：推荐内容是结构化卡片，流式 markdown 体验不如一次性结构化数据。5 张卡片秒级返回，不需要流式
3. **多轮 history 前端也存一份**：双保险——前端拼历史发后端 + 后端 ChatMemory，任一可用
4. **识别保持非流式**：本来就是一次性 JSON，只改 UI 不改协议
