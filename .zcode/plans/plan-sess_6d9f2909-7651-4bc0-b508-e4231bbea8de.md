# 地图功能实现计划（精灵分布 + 交互地图）

## 调研结论

- **精灵分布数据**:页面级 `{{精灵信息/兼容|分布地区=地名1/地名2/...}}`,65% 精灵有结构化地名,与现有 habitat(生态描述)完全不同
- **地图点位数据**:`Data:MapV2/type/{id}/json` 有 11 种类型(庇护所/传送点/炼金台/副本/4种宝箱/3种任务)的 lat/lng 坐标 + title/desc
- **无官方底图**:坐标是游戏内平面坐标系(范围约 -3000~3000),用网格背景替代底图
- **复用抓取**:skill_pool_fetcher 已逐页抓精灵页面,可顺便提取分布地区,零额外网络开销

## 第 1 步:Scraper 扩展（复用现有抓取）

### 1.1 扩展 skill_pool_fetcher.py 提取分布地区
- `parse_skill_pool` 里从已有 params dict 读 `params.get("分布地区")`,按 `/` 拆分(注意带空格,新写 split 逻辑)
- 过滤非地名描述(如"行踪神秘""行踪有些难以捉摸")
- `fetch_skill_pools` 额外返回 `{pet_slug: [地名...]}` 映射
- `_run_skill_pools` 同时产出 `data/seed/pet_locations.json`(结构 `[{pet_slug, location, source_url}]`,多对多,仿 pet_skill)

### 1.2 新增 map_fetcher.py 抓取地图点位
- 新建 `scraper/src/map_fetcher.py`
- 抓取点位类型元数据(`Data:MapV2/type/json` 解析 markType/markTypeName)
- 逐类型抓取 `Data:MapV2/type/{id}/json` 坐标数据
- 产出 `data/seed/map_points.json`(结构 `[{mark_type, type_name, title, desc, lat, lng}]`)
- main.py 加 `--map` 开关

## 第 2 步:后端（2 个新模块）

### 2.1 精灵分布(pet_location)
- V13 迁移:建 `pet_location` 表(pet_id, location, UNIQUE(pet_id, location)),仿 pet_type
- `PetLocation` 实体 + `PetLocationMapper`(仿 PetTypeMapper,含 physicalDeleteByPet)
- SeedRunner 加 `seedPetLocations`
- PetService 加 `searchByLocation(location, page, size)` + `locationsOf(petId)`
- PetDetailDto 加 `List<String> locations`
- PetController:`GET /api/pets?location=岚语峰`(扩展现有 search)+ `GET /api/locations`(地名聚合列表)

### 2.2 地图点位(map_point)
- V14 迁移:建 `map_point` 表(mark_type, type_name, title, desc, lat, lng)
- `MapPoint` 实体 + `MapPointMapper` + DTO + Service + Controller(`GET /api/map/points?type=&page=&size=`)
- SeedRunner 加 `seedMapPoints`

## 第 3 步:前端（交互地图 + 分布查询）

### 3.1 交互地图组件 `components/MapView.tsx`
- **底图**:纯 SVG 网格背景(坐标网格线 + 原点标记),无依赖外部底图
- **点位渲染**:按 lat/lng 真实坐标在网格上定位,用图标 + 颜色区分 11 种类型(庇护所/传送点/宝箱等)
- **交互**:类型筛选(显示/隐藏某类)、点击点位看 title/desc、平移/缩放(基础 pan/zoom)
- props: `points: MapPoint[]`, client 组件

### 3.2 地图页 `app/map/page.tsx`
- Server Component 预取全量点位(`fetchMapPoints({size: 5000})`)
- 渲染 MapView + 类型筛选器 + 点位统计(各类型数量)

### 3.3 精灵分布增强
- 精灵详情页:栖息地下方加「分布地区」区块(地名 chip 列表,可点击跳转)
- 精灵列表页:筛选器加「按分布地区」(下拉/搜索)
- 新增 `app/locations/page.tsx`:地名列表(每个地名显示精灵数 + 点击查看该地区精灵)
- types/pet.ts 加 `locations?: string[]`
- lib/api.ts 加 fetchLocations / fetchPets({location})

### 3.4 导航 + 首页
- site-header:工具组加「地图」入口
- 首页 FEATURE_GROUPS:加地图入口

## 第 4 步:验证

- scraper:抓取 pet_locations.json + map_points.json,验证数据
- 后端:entity/建表/seed key 对齐
- 前端:tsc + 地图点位渲染正确性

## 实施顺序

1. scraper 扩展(skill_pool_fetcher 加分布地区 + 新 map_fetcher)→ 跑抓取
2. 后端 V13+V14 + 2 套模块 + SeedRunner
3. 前端 MapView 交互组件 + 地图页 + 分布增强
4. 导航/首页
5. 验证

## 风险与应对

- **地图点位抓取耗时**:11 种类型,每种一页,约 15 秒
- **分布地区抓取**:复用 skill_pool_fetcher,需重跑(11 分钟)——或用已有 fixture 验证逻辑后全量跑
- **交互地图性能**:点位预估几百个,SVG 渲染无压力
- **底图缺失**:用坐标网格替代,标注"数据来自 BWIKI 社区,坐标系为游戏内坐标"
