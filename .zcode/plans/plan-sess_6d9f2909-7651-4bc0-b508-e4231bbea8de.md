# 道具图鉴全栈实现计划

## 背景与调研结论

参考 BWIKI 洛克王国手游 wiki 的功能板块,SpiritDex 缺少「道具图鉴」。调研已确认:
- **数据可行**:Category:道具 共 **1780 页**(1779 有效),几乎无噪声,`{{物品信息}}` 模板字段高度一致(9 字段,物品名称/稀有度/主分类/次分类/用途/描述/来源/icon/道具版本)
- **图标复用**:`Special:FilePath/{icon}.png` 模式,与现有 petHeadUrl 同链路
- **代码模板**:skills 模块是完整范本,纯展示模块可显著精简(无关联表、无自定义 SQL、无嵌套 DTO)
- **枚举值**:主分类 8 类(材料/技能石/重要/精灵蛋/精灵果实/任务/家具/咕噜球),稀有度 4 级(紫/蓝/橙/绿)

本轮:**全栈贯通道具图鉴**,后续任务/印记板块复制此模板即可。

## 第 1 步:Scraper 抓取层(新增 item_fetcher.py + 命令行开关)

### 1.1 新增 `scraper/src/item_fetcher.py`

仿 `skill_pool_fetcher.py` 模式(独立模块、页面级抓取),核心逻辑:
- `fetch_items(api) -> list[dict]`:分页抓 `Category:道具` 全部成员(cmlimit=500,cmtype=page 过滤掉模板页),逐页抓 wikitext
- 解析 `{{物品信息}}` 模板参数:**复用 `skill_pool_fetcher._extract_template_params` 的同款逐行正则** `^\s*\|([^=]+?)=(.*)$`(模板不嵌套,直接复用)
- 转成 seed items,字段映射:
  ```
  物品名称→name, 稀有度→rarity, 主分类→main_category, 次分类→sub_category,
  用途→usage, 描述→description, 来源→source, icon→icon_id, 道具版本→data_version
  ```
- slug 生成:`item-{序号:04d}`(按抓取顺序,稳定;catalog_id 用 BWIKI 页面名)
- 每条带 `source_url = api.page_url(页面名)`
- 错误隔离:单页失败不阻断(仿 skill_pool_fetcher 的 try/except)

### 1.2 `scraper/main.py` 加 `--items` 分支

仿 `--skill-pools` / `--activities` 独立分支:
- 新增 `--items` flag + `--items-limit N`(调试用)
- `_run_items(args)` 函数:抓取 → 写 `data/seed/items.json`(沿用 `{meta, items}` 结构)

### 1.3 执行抓取(现在跑)

`python main.py --items`,约 30 分钟(QPS≤1,1780 页)。产出 `data/seed/items.json`。

## 第 2 步:后端(新增 item 模块全套,7 个文件 + 迁移)

### 2.1 数据库迁移 `V10__item_schema.sql`

仿 skill 表,纯展示无关联:
```sql
CREATE TABLE IF NOT EXISTS item (
    id            BIGSERIAL    PRIMARY KEY,
    slug          VARCHAR(64)  NOT NULL UNIQUE,    -- item-0001
    catalog_id    VARCHAR(128) NOT NULL UNIQUE,    -- BWIKI 页面名
    name          VARCHAR(128) NOT NULL,
    rarity        VARCHAR(16),                      -- 紫/蓝/橙/绿
    main_category VARCHAR(32),                      -- 材料/技能石/重要/...
    sub_category  VARCHAR(64),
    usage_text    TEXT,
    description   TEXT,
    source_text   TEXT,
    icon_id       VARCHAR(64),
    data_version  VARCHAR(16),
    source_url    VARCHAR(512),
    deleted       SMALLINT     NOT NULL DEFAULT 0,
    created_at    TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMP    NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_item_name          ON item (name);
CREATE INDEX idx_item_main_category ON item (main_category);
CREATE INDEX idx_item_rarity        ON item (rarity);
```

### 2.2 Entity `entity/Item.java`

仿 `Skill.java`:`@TableName("item")` + 字段映射 + `@TableLogic deleted`。注:`catalog_id` 用页面名(可能含中文/符号),长度给 128。

### 2.3 Mapper `mapper/ItemMapper.java`

空接口 `extends BaseMapper<Item>`(纯展示,无自定义 SQL)。

### 2.4 DTO `dto/ItemListItemDto.java` + `dto/ItemDetailDto.java`

- ListDto(卡片用):`slug, name, rarity, mainCategory, subCategory, iconId`
- DetailDto(详情用):全字段,**无嵌套关联**(纯展示,去掉 skills 的 LearnerPet)

### 2.5 Service `service/ItemService.java`

仿 `SkillService` 精简版:
- `extends ServiceImpl<ItemMapper, Item>`,**不注入任何额外 Mapper**
- `search(mainCategory, rarity, q, page, size)`:LambdaQueryWrapper 按 mainCategory/rarity 筛选 + name like,默认按 `id` 排序(道具无 catalog_num 连续编号,用 id)
- `getDetail(slug)`:getOne by slug,空抛 BizException(4040)

### 2.6 Controller `controller/ItemController.java`

仿 `SkillController`:
- `@RequestMapping("/api/items")`
- `GET /api/items?mainCategory=&rarity=&q=&page=1&size=24`
- `GET /api/items/{slug}`

### 2.7 SeedRunner 加 `seedItems`

仿 `seedSkills`:
- `readItems(dir, "items.json")`
- upsert by `catalogId`(BWIKI 页面名)
- 在 `run()` 末尾调用(无下游依赖,位置任意;放 articles 之前)
- 加注入 `private final ItemMapper itemMapper;`

## 第 3 步:前端(新增 item 页面全套,7 个文件 + 导航)

### 3.1 类型 `types/item.ts`

```ts
export interface ItemListItem {
  slug: string; name: string; rarity?: string;
  mainCategory?: string; subCategory?: string; iconId?: string;
}
export interface ItemDetail extends ItemListItem {
  usageText?: string; description?: string; sourceText?: string;
  dataVersion?: string; sourceUrl?: string;
}
```

### 3.2 API `lib/api.ts` 加三段

`ItemFilter` / `fetchItems(filter)` / `fetchItemDetail(slug)`,照抄 skill 的 filter→URLSearchParams 模式,路径 `/api/items`。

### 3.3 图片 helper `lib/image.ts`

加 `itemIconUrl(iconId)`:同构 `Special:FilePath/{iconId}.png`。

### 3.4 组件 `components/ItemCard.tsx` + `components/ItemFilters.tsx`

- **ItemCard**:仿 SkillCard,但加图标(`<Image src={itemIconUrl} unoptimized />`)+ 稀有度配色徽章(紫/蓝/橙/绿,新写一个简单的 `RARITY_COLOR` 映射,仿 type-colors)。卡片壳样式 class 与 SkillCard 一致。
- **ItemFilters**:client 组件,仿 SkillFilters。筛选项:主分类 chip 组(8 枚举,前端硬编码常量 `MAIN_CATEGORIES`)+ 稀有度 chip 组(4 枚举 `RARITIES`)+ 搜索框。URL 驱动,改筛选重置 page。

### 3.5 页面 `app/items/page.tsx` + `app/items/[slug]/page.tsx`

- **列表页**:server component,抄 skills/page.tsx 骨架。读 searchParams(mainCategory/rarity/q/page)→ fetchItems → grid 渲染 ItemCard + Pagination。**无 Suspense 包 types**(道具筛选不需要属性枚举,常量内置)。
- **详情页**:server component,抄 skills/[slug]/page.tsx。`generateStaticParams` 预生成(但 1779 条 SSG 可能慢,**改用动态渲染**:去掉 generateStaticParams + dynamicParams=false,用 `export const dynamic = 'force-static'` 或默认。实际上详情页靠 ISR/动态即可,数据量小)。展示:图标 + 名称 + 稀有度/分类徽章 + 用途/描述/来源段落。

### 3.6 导航 `components/site-header.tsx`

在 `label: "图鉴"` 组加 `{ href: "/items", icon: Package, label: "道具图鉴" }`,import `Package` from lucide-react。

## 实施顺序

1. **scraper**:写 item_fetcher.py + main.py 加 --items → 跑全量抓取(30 分钟,后台)→ 产出 items.json
2. **后端**(抓取进行中并行写):迁移 V10 + Entity/Mapper/DTO/Service/Controller + SeedRunner.seedItems
3. **前端**:类型 + api + image helper + Card/Filters + 列表/详情页 + 导航
4. **验证**:items.json 抽查 + 后端编译 + 前端 build

## 不做的事(本轮范围外)

- 不做任务图鉴、印记图鉴(下轮复制此模板)
- 不动现有 skills/pets 模块
- 不做道具的关联(比如"哪些精灵掉落此道具"——BWIKI 来源字段是自由文本,非结构化)
- 不改部署文档(纯功能迭代)

## 风险与应对

- **抓取 30 分钟**:后台运行,期间并行写后端/前端代码
- **icon 填充率 ~68%**:部分老页无图标,前端 `itemIconUrl` 返回 null 时显示占位符
- **1779 条 SSG 慢**:详情页用动态渲染而非全量 SSG,避免构建超时
